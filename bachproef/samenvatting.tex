\chapter*{\IfLanguageName{dutch}{Samenvatting}{Abstract}}



Deze bachelorproef onderzoekt in welke mate verschillende large language models betrouwbare en technisch correcte Python code genereren voor representatieve data analyse en ETL taken binnen de context van studenten Toegepaste Informatica aan HOGENT. Het onderzoek vertrekt vanuit de toenemende inzet van generatieve AI in programmeeropdrachten, waarbij studenten code laten genereren voor data verwerking, transformatie en automatisatie, maar waarbij niet altijd duidelijk is hoe correct, veilig en onderhoudbaar die output werkelijk is. De centrale onderzoeksvraag luidt daarom: in welke mate verschilt de technische correctheid en betrouwbaarheid van Python code die wordt gegenereerd door verschillende LLM's voor representatieve data analyse en ETL taken binnen de context van HOGENT studenten Toegepaste Informatica, en welke praktische richtlijnen kunnen studenten helpen om foutieve of suboptimale output te herkennen en te verhelpen?

Om deze vraag te beantwoorden werd een vergelijkende benchmark opgezet met vijf modellen, namelijk deepseek coder v2, gemma 4, llama 3, mistral en qwen 2.5 coder. Elk model werd getest op 20 representatieve taken voor data analyse en ETL. De gegenereerde code werd geëvalueerd op syntaxis, beveiliging, uitvoerbaarheid, performantie, codekwaliteit en functionele correctheid. De evaluatie combineerde automatische checks, waaronder syntaxiscontrole, security analyse, execution tests, performance metingen en metriek gebaseerde codeanalyse, met functionele testen die controleerden of de output van de code overeenkwam met de verwachte resultaten.

De resultaten tonen duidelijke verschillen tussen de modellen. Qwen 2.5 coder presteerde het best met een overall pass rate van 65 procent, gevolgd door deepseek coder v2 met 55 procent. Llama 3 behaalde 45 procent, terwijl mistral en gemma 4 duidelijk zwakker scoorden met respectievelijk 20 procent en 15 procent. Opvallend is dat meerdere modellen wel vaak syntactisch correcte code genereerden, maar toch faalden op functionele correctheid of op specifieke kwaliteitscontroles. De resultaten wijzen er dus op dat een codefragment dat er op het eerste zicht correct uitziet, nog steeds inhoudelijke fouten, veiligheidsproblemen of performantieproblemen kan bevatten.

De meerwaarde van dit onderzoek ligt in de combinatie van een domeinspecifieke vergelijking van LLM's en een praktisch bruikbaar evaluatiekader voor studenten en docenten Toegepaste Informatica. De bevindingen tonen aan dat generatieve AI vandaag nuttig kan zijn als ondersteunende programmeerhulp, maar niet zonder menselijke controle mag worden gebruikt in data analyse en ETL contexten. Op basis van deze bachelorproef kunnen studenten gerichter leren beoordelen wanneer LLM gegenereerde code bruikbaar is, waar de grootste risico's zitten en welke controles noodzakelijk zijn om fouten te detecteren en te verbeteren.
